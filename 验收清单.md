# Gear Metrics 验收清单（v1）

## 1. 环境准备

- [ ] 已复制 `.env.example` 到 `.env.local`
- [ ] 已配置 `NEXT_PUBLIC_SUPABASE_URL`
- [ ] 已配置 `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- [ ] 已执行 Supabase SQL v1（建表、索引、RLS）
- [ ] 至少有 1 个可用用户会话用于 RLS 验证

## 2. 核心流程验收（录入 -> 筛选 -> 推荐 -> 审计）

### 2.1 装备录入与编辑

- [ ] 在 `/gears` 新增装备成功（名称、品牌、分类、温标、状态）
- [ ] 编辑任意装备字段后保存成功
- [ ] 温标校验生效（`min_temp > max_temp` 时给出中文报错）
- [ ] 删除装备成功，列表刷新
- [ ] 状态切换按钮可循环切换（在用 -> 在途 -> 闲置 -> 损耗）

### 2.2 筛选

- [ ] 分类筛选生效（仅显示目标分类）
- [ ] 状态筛选生效（仅显示目标状态）
- [ ] 分类+状态组合筛选生效

### 2.3 CSV 导入

- [ ] 上传 Notion CSV 后可看到预览
- [ ] 缺少必填字段映射时，阻止导入
- [ ] 导入完成后显示成功/失败条数
- [ ] 失败行可看到“行号 + 原因”

### 2.4 AI 三层穿衣推荐

- [ ] 输入目的地/活动/温标后可生成结构化结果
- [ ] 预筛选仅使用 `status=在用` 且相关品类
- [ ] 缺层时返回“当前装备库无法覆盖该温标，缺少XX层”
- [ ] 输出包含排汗层/保暖层/防风防雨层、下装、鞋履

### 2.5 AI 新购审计（中立建议）

- [ ] 输入拟购信息后返回 `purchaseIndex(1-10)`
- [ ] 返回功能重叠、预算溢价、系统互补三个维度
- [ ] 输出结论为中立风格（建议购买/谨慎购买/暂不购买）
- [ ] 输出中包含 `reasons` 与 `neutralAdvice`

## 3. 技术校验

- [x] `npm run lint` 通过
- [x] `npm run build` 通过
- [ ] 本地 `npm run dev` 启动后 `/` 和 `/gears` 页面可访问

## 4. 已知限制（v1）

- 当前未集成完整 Supabase Auth 登录流；RLS 生效依赖实际会话注入。
- CSV 解析为轻量实现，复杂多行引号场景覆盖有限。
- AI 能力依赖 `GEMINI_API_KEY` 或 `ANTHROPIC_API_KEY`，未配置时会直接返回错误。
- 暂未引入自动化测试框架（当前以手工验收为主）。
