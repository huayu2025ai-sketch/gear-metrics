# Gear Metrics v1 交付文档

## 1. 版本范围

- 已交付：装备 CRUD、分类/状态筛选、CSV 导入（Notion 映射）、AI 三层穿衣推荐、AI 新购审计（中立建议）。
- 已完成校验：`npm run lint`、`npm run build`。

## 2. 环境变量

在项目根目录创建 `.env.local`：

```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
GEMINI_API_KEY=your_gemini_api_key
ANTHROPIC_API_KEY=your_anthropic_api_key
```

说明：
- 推荐至少配置一个 AI Key（Gemini 或 Anthropic）。
- 未配置 AI Key 时，AI 功能会返回中文错误提示。

## 3. 数据库初始化


## 6. 常见故障排查

1. AI 接口报未配置 Key  
- 检查 `.env.local` 中 `GEMINI_API_KEY` / `ANTHROPIC_API_KEY`。

2. 装备写入失败（RLS/权限错误）  
- 检查是否有有效 Supabase 用户会话。  
- 检查 RLS 是否执行了 `insert/update/delete` 策略。

3. CSV 导入失败行过多  
- 检查表头映射是否符合预期（`name/brand/category/min_temp/max_temp`）。  
- 检查温标是否合法且 `min_temp <= max_temp`。

4. 构建时字体网络失败  
- 当前使用 `next/font/google`，在受限网络环境可能拉取失败。  
- 可改成本地字体以消除外网依赖。

## 7. 后续演进建议（v2+）

3. 引入自动化测试（单元 + 集成 + E2E）。  
4. AI 上下文优化：增加 SQL 召回策略，后续引入 embedding 检索。  
5. 逐步支持分类可扩展模型（固定枚举 -> 分类表）。
